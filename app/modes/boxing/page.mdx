# Room: Boxing

Boxing is the teacher/student room mode. It works like a live oral check: the teacher asks, the student answers, and follow-ups continue naturally in chat.

![Room Boxing terminal example using hello-quiz.md](/mode-boxing-terminal.svg)

## Start a room

```bash
quizmd room --create --mode boxing --role teacher --quiz hello-quiz.md
```

Students join with the room name and token printed by the teacher:

```bash
quizmd room --join <room-name> --token <room-token> --role student
```

## Rules

- Teacher asks, student answers, and follow-ups continue naturally.
- Use chat for conversation; teacher can set score with `/score <0-100>`.
- Either teacher or student can end session with `/end`.
- Transcript and final score are saved automatically at session end.
- Press `Ctrl+C` to exit at any time.

## Best for

- One-to-one assessment.
- Viva-style coding checks.
- Guided feedback sessions.

## Example session prompt

```md
## Question 1
Explain why this function is risky:

`def add_item(item, bucket=[]): bucket.append(item); return bucket`

- Mutable default arguments are shared between calls
- Lists cannot be returned from functions
- `append` creates a new list every time
- Function arguments cannot have defaults

Answer: 1
Type: single
Explanation: The same default list is reused, so later calls can unexpectedly contain earlier values.
```
