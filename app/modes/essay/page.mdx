# Essay Mode

Essay Mode is structurally different from MCQ mode.

![Essay Mode terminal example from hello-essay.md](/mode-essay-terminal.svg)

## What it does

- Uses a Markdown-defined essay prompt and rubric.
- Collects essay text in editor flow.
- Evaluates against rubric and returns score + feedback sections.
- Supports Gemini, OpenAI, and Anthropic through provider-specific API keys.

## Example prompt

```md
# Essay Question: Review dependency pinning

## Question
Explain why a Python team might pin package versions in `requirements.txt`.

## Instructions for Students
Write a short technical answer for a junior developer joining the project.

## Evaluation Criteria (Total: 3 points)
1. **Reproducible installs (1 point)**
- Explains that pinned versions make installs predictable
2. **CI/team consistency (1 point)**
- Mentions consistent environments across machines or CI
3. **Maintenance tradeoff (1 point)**
- Notes that pins still need updates for bug fixes and security patches

## Reference Answer
Pinned dependencies make installs predictable across machines and CI, but they need regular review so security fixes are not missed.

## AI Evaluation Rules
Evaluate only against the criteria above. Give partial credit for accurate but incomplete answers.

## Output Format
Score: XX%

Feedback:
- What worked
- What is missing
- One suggestion
```

## Key differences from MCQ

- Starts with `# Essay Question: ...`.
- Requires dedicated sections (`Question`, `Instructions`, `Evaluation Criteria`, etc.).
- Uses AI provider settings (`--ai-provider`, `--ai-model`, `--ai-timeout`) when enabled.

## Run

```bash
export GEMINI_API_KEY="your_key_here"
quizmd --validate hello-essay.md
quizmd hello-essay.md
```

Windows PowerShell:

```powershell
$env:GEMINI_API_KEY="your_key_here"
quizmd hello-essay.md
```

See full provider setup on [Essay Mode & AI Providers](/essay-ai-providers).
